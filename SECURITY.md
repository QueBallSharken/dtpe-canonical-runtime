# Security Policy

## Reporting a Vulnerability

If you discover a security issue in the DTPE runtime, please report it
responsibly.

Open a GitHub issue describing the problem and the conditions under
which it occurs.

Include:

- a description of the issue
- steps to reproduce
- the expected behavior
- the observed behavior

Please do not publicly disclose vulnerabilities before maintainers
have had an opportunity to review and address them.

## Scope

The DTPE runtime is currently a deterministic governance prototype.

The project demonstrates:

- policy-governed execution
- cryptographic authority binding
- canonical receipt generation
- append-only ledger evidence
- independent offline verification

The system is not currently a production distributed system and should
be evaluated as a research / prototype runtime.
